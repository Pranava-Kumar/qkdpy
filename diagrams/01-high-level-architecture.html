Key Design Decisions:

Decision

Choice

Rationale

State representation

Statevector (not density matrix)

Simpler, faster for pure states; density matrix computed on demand

Randomness

CSPRNG (secrets module)

All cryptographic randomness uses secure entropy; never numpy.random for crypto

Error handling

Either<ClientException, T> not used (pure Python)

Exceptions propagate naturally; no Arrow dependency

Protocol orchestration

BaseProtocol.execute() template method

Consistent flow across all protocols: prepare → transmit → measure → sift → EC → PA

Integrations

Separate wrappers, not inheritance

Each framework (Qiskit, PennyLane) wrapped independently; no coupling

Noise simulation

Quantum trajectory (statevector unraveling)

Kraus operators applied stochastically; more efficient than full density matrix

Key management

Modular pipeline

EC and PA are independent, composable stages
